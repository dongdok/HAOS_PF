# 자기소개서 1페이지 — HAOS_Control

## 1. 한 줄 소개
저는 Home Assistant OS 기반으로 집 전체를 로컬 우선 아키텍처로 재설계하고, 자동화 운영 데이터를 정량적으로 관리하는 스마트홈 엔지니어입니다.

## 2. 문제 인식
기존 환경은 SmartThings/Tuya/iOS 단축어가 분리 운영되어 다음 문제가 있었습니다.
- 플랫폼 파편화로 전체 흐름 추적이 어려움
- 클라우드 의존으로 장애 시 제어 안정성 저하
- 엔티티 명명 불일치로 자동화/대시보드 유지보수 비용 증가

## 3. 해결 접근
저는 `HAOS_Control` 프로젝트에서 아래를 직접 설계/운영했습니다.
- LocalTuya + Zigbee2MQTT + MQTT + SmartThings 혼합 통합
- 캐노니컬 엔티티 표준화 및 재등록 복구 절차 확립
- 추천 엔진(로그 수집 → 패턴 탐지 → 후보 제안 → 피드백 반영) 구축
- 운영 대시보드/스크립트/자동화를 분리해 아키텍처 일관성 유지

- 충돌 피드백: **6건** (`intended 5 / unintended 1`)

## 5. 기술적으로 어려웠던 지점과 해결
- Zigbee 채널 변경/기기 재등록 후 IEEE 기반 엔티티 드리프트가 발생
- 기기별로 디바이스명, 엔티티명, 자동화 참조를 한 번에 복구하는 표준 절차를 적용
- `unknown/unavailable`를 임시값으로 덮지 않고 실패를 명시 기록하여 데이터 신뢰성 유지

## 6. 제가 만드는 가치
저는 기능 추가보다 먼저 **운영 가능한 구조**를 만듭니다.  
즉, “돌아가는 코드”가 아니라 “장애가 나도 복구 가능한 시스템”을 설계하고, 그 결과를 로그와 지표로 증명합니다.
