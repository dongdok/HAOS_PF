# 공개 배포 체크리스트

## 1) 민감정보 제거

- [ ] 내부 IP 대역 전체 마스킹 (`192.168.x.x` 형태)
- [ ] MAC/IEEE 전체 또는 일부 마스킹
- [ ] 토큰/API Key/비밀번호/웹훅 URL 제거
- [ ] 개인 식별 정보(실명/정확 주소/디바이스 고유명) 제거

## 2) 운영 보안

- [ ] `secrets.yaml` / `.env` / 인증서 / 백업파일 미포함 확인
- [ ] 실제 HA base_url, VPN 도메인, 외부 접근 URL 비공개 처리
- [ ] 스크린샷 메타데이터(계정/알림/탭 정보) 점검

## 3) 기술 검증 가능성 유지

- [ ] 아키텍처/운영 흐름은 재현 가능한 수준으로 설명
- [ ] 장애 사례는 증상-원인-조치-결과로 작성
- [ ] 수치 지표는 정의 포함(과장 방지)

## 4) 제출용 품질

- [ ] 문서 링크 깨짐 없음
- [ ] 다이어그램 렌더링 확인
- [ ] 용어 통일 (엔티티/통합구성요소/서비스 정의 일치)
- [ ] 기준 날짜/시간 명시

## 5) GitHub 공개 전략

- [ ] 공개 저장소에는 문서/다이어그램 중심으로 구성
- [ ] 실제 운영 코드/설정 원문은 private 유지
- [ ] 필요 시 "요청 시 private 접근 초대" 문구 추가
