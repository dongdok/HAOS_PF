# 자동화 사례 (네트워크 운영 관점)

## 사례 A) 밤 시간 거실 조명 OFF

- 입력 조건
  - 거실 재실 해제
  - 시간대: 23:00~06:00
  - 타 구역(주방/화장실/옷방) 미재실

- 출력 동작
  - 거실 라인 조명 OFF
  - 거실 스탠드 조명 OFF

- 네트워크 관점 의미
  - 이동 동선 이벤트를 단일 룰로 처리하면서도 오탐 줄임
  - 중복 명령 최소화로 무선 구간 부하 억제

## 사례 B) 화장실 재실/도어 연동 조명/환풍

- 입력 조건
  - 화장실 재실 감지 또는 도어 상태 변화

- 출력 동작
  - 조명 ON/OFF
  - 체류 시간 기반 환풍 타이머 동작

- 네트워크 관점 의미
  - 이벤트 기반 제어로 배터리/저전력 센서 운영 최적화
  - 상태 확인 조건으로 timeout 가능성 감소

## 사례 C) 현관 도어 기반 복귀 시나리오

- 입력 조건
  - 현관 도어 이벤트 + 가상 스위치 상태 조합

- 출력 동작
  - 복귀 자동화 실행(조명/기기 시나리오)

- 네트워크 관점 의미
  - 단일 센서 이벤트를 즉시 실행하지 않고 상태 머신으로 안정화
  - 외부 의존 경로가 불안정할 때도 로컬 경로 우선 유지

## 공통 설계 규칙

1. 트리거와 조건 분리
- 이벤트는 넓게 받고, 실행은 조건으로 강하게 제한

2. 이미 같은 상태면 명령 생략
- ON 대상이 이미 ON이면 전송하지 않음

3. 재등록 친화 구조
- 디바이스 교체/재조인 시 참조만 바꾸면 동작 복구 가능
