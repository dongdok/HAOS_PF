# 자동화 사례 (운영 포인트)

상세 생활 시나리오: [실사용 자동화 예시](automation_real_world_flows.md)

## 1. 밤 시간 거실 조명 자동 끄기

- **조건**
  - 거실에 사람이 없고, 밤 11시부터 새벽 6시 사이일 때
  - 주방이나 옷방 등 다른 구역에도 사람이 없을 때
- **동작**
  - 거실의 모든 조명을 끔
- **운영 포인트**
  - 이동 동선을 고려해 사람이 완전히 없는 것이 확인될 때만 동작하게 하여 오동작을 줄임
  - 이미 꺼져 있는 경우 중복 명령을 보내지 않아 네트워크 부하를 줄임

## 2. 화장실 재실 연동 조명 및 환풍기

- **조건**
  - 화장실 문이 열리거나 사람이 감지될 때
- **동작**
  - 조명을 바로 켜고, 머무는 시간이 길어지면 환풍기를 가동함
- **운영 포인트**
  - 배터리 방식 센서의 전력 소모를 줄이기 위해 필수적인 이벤트만 기록하게 설정
  - 통신 지연으로 인해 명령이 누락되지 않도록 상태 확인 로직을 추가

## 3. 현관문 기반 귀가 시나리오

- **조건**
  - 현관문이 열리고 특정 버튼이나 상태가 확인될 때
- **동작**
  - 복귀 전용 자동화(조명 및 가전)를 순차적으로 실행
- **운영 포인트**
  - 센서 신호 하나에 바로 반응하지 않고, 집안의 전체적인 상태를 보고 판단하게 하여 안정성 확보
  - 외부망이 불안정해도 단지 내부망 안에서 모든 동작이 완결되도록 구성

## 공통 설계 규칙

1. **트리거와 조건의 분리**: 이벤트는 넓게 받되, 실제 실행은 꼼꼼한 조건 확인 후에 진행합니다.
2. **중복 명령 방지**: 이미 목표 상태(예: 조명 ON)라면 같은 명령을 다시 보내지 않습니다.
3. **유지보수 고려**: 기기가 고장 나거나 교체되어도 이름만 똑같이 맞춰주면 바로 복구되도록 설계했습니다.
