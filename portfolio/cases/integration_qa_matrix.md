# 연동 테스트 항목

> 목적: 기기 간 연동 결과를 동일한 기준으로 확인하고 기록

| 시나리오 | 테스트 내용 | 기대 결과 | 확인 방법 | 판정 |
|---|---|---|---|---|
| 조명 켜기 연동 | 스위치에서 조명 켬 | 서버 및 앱 상태가 '켜짐'으로 일치 | 로그 및 앱 화면 비교 | Pass/Fail |
| 조명 끄기 연동 | 앱에서 조명 끔 | 실제 조명이 꺼지고 상태 일치 | 조명 동작 및 로그 확인 | Pass/Fail |
| 센서 감지 반영 | 문 열림/사람 감지 발생 | 서버가 이벤트를 받고 상태를 즉시 반영 | 기록된 시각과 상태 비교 | Pass/Fail |
| 지연 대응 확인 | 통신이 느린 상황 재현 | 재시도 로그 기록 및 정상 복구 여부 | 통신 오류 로그 확인 | Pass/Fail |
| 조치 후 재검증 | 장애 해결 후 재실행 | 문제없이 동일 시나리오가 동작함 | 2회 이상 반복 테스트 | Pass/Fail |

## 결함 처리 및 관리 규칙

- 결함 발견 시 **시각, 대상 기기, 재현 순서, 로그, 영향 범위**를 필수로 기록합니다.
- 수정 완료 후에는 반드시 동일한 조건에서 다시 테스트합니다.
- 임시방편보다 기기 이름(참조)이나 구조적인 정합성을 복구하는 것을 우선합니다.
