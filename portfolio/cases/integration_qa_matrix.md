# 3자 연동 QA 매트릭스 (월패드-서버-플랫폼)

> 목적: 연동 경로별 검증 결과를 동일 형식으로 기록

| 시나리오 | 입력/트리거 | 기대 결과 | 확인 방법 | 판정 |
|---|---|---|---|---|
| 조명 ON 연동 | 월패드에서 조명 ON | 서버/앱 상태 ON 동기화 | 월패드 상태, 서버 로그, 앱 UI 비교 | Pass/Fail |
| 조명 OFF 연동 | 앱에서 조명 OFF | 월패드/서버 상태 OFF 동기화 | 앱 실행 이력, 서버 반영, 월패드 확인 | Pass/Fail |
| 센서 이벤트 반영 | 도어/재실 센서 이벤트 발생 | 플랫폼 이벤트 수신 및 상태 반영 | 이벤트 시각 비교, 로그 확인 | Pass/Fail |
| 예외 응답 | 네트워크 지연 상황 | 재시도 또는 오류 기록 | timeout/delivery fail 로그 확인 | Pass/Fail |
| 복구 확인 | 장애 조치 후 재실행 | 동일 시나리오 정상 동작 | 2회 반복 테스트 | Pass/Fail |

## 결함 처리 규칙

- 결함 등록 시 필수 항목: 시각, 대상, 재현 단계, 로그, 영향 범위
- 수정 반영 후 동일 케이스 재검증
- 임시 우회 대신 참조/구조 정합성 복구 우선
