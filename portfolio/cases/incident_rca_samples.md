# 장애 분석 사례

## 사례 01. 지그비 명령 전송 지연

- **증상**: 명령 전송 실패 및 응답 지연 반복
- **원인**: 와이파이 전파 간섭이 심한 구역에서 지그비 통신 경로가 불안정해짐
- **조치**: 통신 채널 조정 및 기기를 다시 네트워크에 조인(Join) 시켜 경로 재설정
- **확인**: 동일 제어 시나리오 반복 실행 시 실패 빈도가 현저히 낮아진 것을 확인

## 사례 02. 기기 재등록 후 자동화 멈춤

- **증상**: 기기는 정상 연결되었으나 자동화가 대상을 찾지 못함
- **원인**: 기기 재등록 과정에서 이름(ID)이 바뀌어 기존 자동화가 옛날 이름을 참조함
- **조치**: 표준 명칭 규칙에 따라 기기 이름을 다시 맞추고 전체 연결 정보 동기화
- **확인**: 자동화 실행 기록에서 대상 기기를 정상적으로 찾고 실행되는지 확인

## 사례 03. 동일 명령 중복 전송

- **증상**: 같은 켜기/끄기 명령이 한꺼번에 여러 번 전송됨
- **원인**: 현재 기기의 상태를 확인하지 않고 무조건 명령을 보내는 로직 설계
- **조치**: "이미 목표 상태와 같으면 실행을 건너뛰는" 조건 추가
- **확인**: 로그상에서 중복 전송되는 데이터가 사라지고 응답 속도가 개선된 것을 확인
