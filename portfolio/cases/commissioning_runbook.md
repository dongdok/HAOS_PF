# 시운전(Commissioning) 런북 (공개용)

> 목적: 신규 단지/기기 적용 전 통신, 제어, 연동 상태를 동일한 기준으로 점검

## 1) 사전 점검

1. 네트워크
- 대상 기기 IP/DHCP 예약 상태 확인
- 내부망 대역 및 게이트웨이 확인

2. 제어 경로
- Zigbee: Coordinator 연결 상태, MQTT 브로커 연결 확인
- Wi-Fi: 로컬 제어(LocalTuya 등) 연결 상태 확인

3. 식별 정합성
- 디바이스명, 엔티티 ID, 자동화 참조 대상 일치 여부 확인

## 2) 기능 점검

1. 기본 제어
- ON/OFF, 상태 조회, 앱 반영 시간 확인

2. 연동 점검
- 트리거(센서) → 자동화 → 대상 기기 동작까지 전체 경로 확인

3. 로그 점검
- timeout, delivery fail, unavailable 이벤트 확인

## 3) 장애 발생 시 처리

1. 증상 기록
- 발생 시각, 대상 엔티티, 실패 로그 문자열 기록

2. 원인 분리
- 네트워크/기기/참조 불일치 중 우선 원인 분류

3. 조치
- 재조인, 채널 조정, 참조 동기화(자동화/스크립트/대시보드)

4. 재검증
- 동일 시나리오 2회 이상 반복 테스트

## 4) 완료 기준

- 핵심 시나리오 통과
- 주요 로그 오류 미발생
- 운영 문서와 실제 엔티티 일치
