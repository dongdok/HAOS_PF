# 현장 점검 가이드

> 목적: 신규 기기 등록 시 통신 및 제어 상태를 동일한 기준으로 확인

## 1. 사전 확인

1. **네트워크 상태**
- 기기에 고정 IP가 잘 할당되었는지 확인
- 내부망 게이트웨이 연결 상태 확인

2. **제어 경로 확인**
- 지그비: 코디네이터 및 메시지 브로커(MQTT) 연결 상태 확인
- 와이파이: 로컬 제어 기능 연결 상태 확인

3. **이름 정합성**
- 기기 이름, 식별자(ID), 자동화 연결 대상이 일치하는지 확인

## 2. 기능 점검

1. **기본 제어**
- 켜기/끄기 명령 시 정상 동작 및 앱 반영 속도 확인

2. **자동화 연동**
- 센서 감지부터 자동화 실행, 기기 동작까지 전체 과정 확인

3. **로그 기록**
- 통신 지연이나 명령 확인 실패 등 오류 로그 발생 여부 확인

## 3. 문제 발생 시 조치

1. **상황 기록**: 발생 시각과 대상 기기, 실패 로그의 핵심 내용을 기록합니다.
2. **원인 분류**: 통신 문제인지, 기기 자체 문제인지, 아니면 이름이 틀렸는지 파악합니다.
3. **조치 단계**: 필요시 기기를 재등록하거나, 통신 채널을 조정하고 이름(참조)을 다시 맞춥니다.
4. **최종 확인**: 동일한 상황을 최소 2회 이상 반복 테스트하여 문제가 해결되었는지 봅니다.

## 4. 완료 기준

- 핵심 자동화 시나리오 통과
- 주요 로그상 오류 미발생
- 운영 문서와 실제 등록된 기기 정보 일치
