# IoT 네트워크 엔지니어 포트폴리오 요약

## 1) 프로젝트 개요

- 프로젝트명: HAOS_Control
- 목표: 클라우드 의존형 스마트홈을 로컬 우선 구조로 재설계해, 장애 시에도 제어/관측 연속성을 확보
- 역할: 네트워크 설계, 통합 운영, 장애 분석/복구, 자동화 표준화, 운영 문서화

## 2) 왜 시작했는가

기존 환경은 SmartThings, Tuya, iOS 단축어가 분산 운영되어 장애 원인 추적이 어려웠습니다.
특히 외부망 상태에 따라 제어 품질이 흔들리는 문제가 있어 Home Assistant 중심 통합으로 전환했습니다.

## 3) 설계 원칙

1. 로컬 제어 우선
- MQTT, Zigbee2MQTT, LocalTuya를 기본 제어 경로로 사용

2. 단일 운영 진실 소스 유지
- 엔티티/디바이스명/자동화 참조를 캐노니컬 기준으로 관리

3. 장애를 숨기지 않음
- fallback 추정값 대신 실패를 실패로 기록하고 원인 분류 후 복구

4. 운영 가능한 자동화
- 중복 명령 전송 방지, 조건 충돌 분리, 로그 기반 개선 루프 유지

## 4) 직무 적합 포인트

- 이기종 프로토콜 통합 운영: Zigbee, MQTT, Wi-Fi, Cloud 연동
- 무선망 장애 대응: 채널 조정, 재조인, 라우팅/타임아웃 분석
- 변경관리: 재등록 이후 엔티티 참조 정합성 복구
- 운영 지표 기반 개선: 성공/취소/실패, 자동/수동 이벤트 비율 관리

## 5) 한 줄 요약

HAOS_Control은 “기능 구현”보다 “네트워크 운영 안정성”에 초점을 둔 프로젝트로,
혼합 IoT 망에서 장애를 재현 가능하게 분석하고 구조적으로 복구한 실무 경험을 증명합니다.
